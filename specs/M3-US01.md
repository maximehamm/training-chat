# Spec technique — M3-US01

> US : "En tant que développeur, je veux rechercher dans la doc avec une requête en langage naturel
> afin de trouver rapidement l'information sans parcourir tous les fichiers."
> — Spec Fonctionnelle v1.1, §4.3

## Endpoints

| Méthode | Path      | Description                                      |
|---------|-----------|--------------------------------------------------|
| POST    | /search   | Recherche par mots-clés dans docs/ avec scoring  |

## Contrat d'API

### Request

`POST /search` — `Content-Type: application/json`

| Champ            | Type    | Requis | Contraintes                        | Défaut |
|------------------|---------|--------|------------------------------------|--------|
| `query`          | String  | Oui    | 1–500 caractères                   | —      |
| `max_results`    | Integer | Non    | 1–50                               | 5      |
| `include_content`| Boolean | Non    | Aucune                             | false  |

### Response

**200 OK**
```json
{
  "results": [
    {
      "title":   "string",
      "path":    "string",
      "score":   0.95,
      "snippet": "string (max 200 caractères)",
      "content": "string (présent uniquement si include_content=true)"
    }
  ],
  "total":         5,
  "query_time_ms": 42
}
```

**400 Bad Request** — query absente ou vide
```json
{ "error": "INVALID_QUERY", "message": "..." }
```

**400 Bad Request** — query > 500 caractères
```json
{ "error": "INVALID_QUERY", "message": "..." }
```

**400 Bad Request** — max_results hors [1, 50]
```json
{ "error": "INVALID_PARAMETER", "message": "..." }
```

## Contraintes

- Retourner `200 { results: [], total: 0, query_time_ms: N }` si `docs/` n'existe pas (ne pas retourner 404).
- Trier les résultats par `score` décroissant.
- `snippet` : max 200 caractères, tronqué proprement (pas en plein mot si possible).
- `content` : inclus dans chaque résultat uniquement si `include_content=true`.
- Indexation récursive de `docs/` : seuls les fichiers `.md` et `.txt` sont traités.
- Fichiers > 1 Mo : ignorés silencieusement (log warning serveur, pas d'erreur client).
- Score calculé par fréquence de termes normalisée (TF).
- Recherche insensible à la casse et aux accents.
- Ne PAS exposer de chemin absolu dans `path` — chemin relatif à `docs/`.
- Ne PAS appeler l'API Claude pour la recherche (recherche locale uniquement).

## Critères de complétion

1. `POST /search` avec `{ "query": "spring boot" }` retourne 200 avec `results[]`, `total`, `query_time_ms`.
2. Les résultats sont triés par `score` décroissant.
3. Chaque `snippet` ne dépasse pas 200 caractères.
4. `POST /search` avec `include_content: true` retourne le champ `content` dans chaque résultat.
5. `POST /search` sans `query` retourne 400 `INVALID_QUERY`.
6. `POST /search` avec `query` de 501 caractères retourne 400 `INVALID_QUERY`.
7. `POST /search` avec `max_results: 0` retourne 400 `INVALID_PARAMETER`.
8. `POST /search` avec `max_results: 51` retourne 400 `INVALID_PARAMETER`.
9. `POST /search` quand `docs/` n'existe pas retourne 200 `{ results: [], total: 0 }`.
10. La recherche est insensible à la casse (`"Spring"` = `"spring"`).
11. Seuls les fichiers `.md` et `.txt` de `docs/` sont indexés (récursivement).
